# Продуктовая и игровая аналитика мобильного приложения: Retention, A/B-тестирование и ивент-метрики

Аналитический проект для разработчика мобильных игр. Включает в себя автоматизацию расчета когортного удержания (Retention Rate), статистическую оценку A/B-теста акционных предложений и проработку продуктовых метрик для внутриигровых событий.

---

## Стек технологий & Библиотеки
* **Язык:** Python 3
* **Обработка данных:** Pandas, NumPy
* **Математическая статистика:** SciPy (`scipy.stats`), Statsmodels
* **Визуализация данных:** Seaborn, Matplotlib
* **Вспомогательные инструменты:** `tqdm` (прогресс-бар для бутстрап-итераций)

## Задача 1. Автоматизация расчета Retention Rate (Когортный анализ)

### Цель:
Написать универсальную и переиспользуемую функцию на Python, которая принимает на вход логи регистраций и авторизаций пользователей, рассчитывает $n$-day Retention и визуализирует когортную таблицу в виде тепловой карты (Heatmap).

### Реализация:
* Перевод временных меток из формата `Unix Timestamp` в дневной формат (`datetime64[D]`).
* Фильтрация данных по задаваемому временному интервалу (`start_date`, `end_date`).
* Объединение логов регистраций и активностей по `uid` с расчетом интервала `days = auth_ts - reg_ts`.
* Агрегация уникальных пользователей по когортам и дням жизни с помощью `.pivot()` и нормализация таблицы относительно 0-го дня.

### Результат:
На выходе генерируется наглядный график sns.heatmap, позволяющий продуктовой команде отслеживать динамику удержания игроков по дням от момента установки.

## Задача 2. Анализ результатов A/B-тестирования акционных предложений

### Цель:
Оценить эффективность нового набора акционных предложений в тестовой группе. Известно, что $ARPU$ в тестовой группе выше на $5.3\%$, однако распределения трат существенно отличаются.

### Исследование структуры данных и выбросов:
* Контрольная группа (A): Распределение строго бимодальное. Присутствует основная масса мелких чеков и сегмент «китов» с чеками до $37\,433$.
* Тестовая группа (B): Чеки распределены более равномерно в диапазоне от $2\,000$ до $4\,000$.

### Статистическое тестирование:
1. Проверка нормальности (Тест Шапиро-Уилка): $p\text{-value} < 0.05$ для обеих выборок. Распределения сильно отклоняются от нормального, применение классического $t$-теста некорректно.
2. Оценка ARPU и ARPPU (Bootstrap):Бутстрап для $ARPU$: $p\text{-value} \approx 0.493$. Статистически значимых различий между группами не обнаружено.Бутстрап для $ARPPU$: $p\text{-value} \approx 0.098$ (при $\alpha = 0.05$). Разница не достигает уровня статистической значимости.
3. Оценка конверсии CR (Критерий Хи-квадрат Пирсона $\chi^2$):$\chi^2 = 4.37$, $p\text{-value} = 0.0365 < 0.05$.Вывод: Падение конверсии в тестовой группе с $0.95\%$ до $0.89\%$ является статистически значимым.

### Итоговый вердикт по A/B-тесту:
Тестовый набор предложений не рекомендуется выкатывать на всех пользователей в текущем виде. Несмотря на точечный рост $ARPPU$, мы фиксируем статистически значимое падение конверсии ($CR$), при этом суммарный прирост $ARPU$ не подтвержден статистически ($p > 0.05$). Необходимо провети повторный A/A и A/B тест с дополнительной сегментацией аудитории (выделив «китов» и платящих игроков).

## Задача 3. Формирование системы метрик для внутриигровых ивентов

### Контекст:
Оценка успешности временных тематических событий (Event-механик) в игре Plants & Gardens.

## 1. Базовый набор метрик для стандартного ивента:
* Финансовые & Конверсионные: Revenue, ARPU / ARPPU, CR в ивентовые покупки.
* Аудиторные & Вовлеченность: DAU / WAU / MAU во время ивента, Retention (D1, D7, D30), New Users / Lost Users.

## 2. Специфические метрики при усложнении механики (проигрыш с откатом на несколько уровней назад):
При введении жестких механик наказывающего характера базовых метрик недостаточно — критически важно отслеживать негативные эффекты и точки оттока:
* Churn Rate (Коэффициент оттока): Для оценки того, сколько игроков покидают игру из-за высокой сложности.
* Воронка прохождения уровней (Level Funnel): Позволяет локализовать конкретный уровень-хардблок, на котором происходит наибольший отвал игроков после сброса прогресса.
* Average Session Length (ASL) & Сессии: Изменение длительности и частоты игровых сессий.
* Repeat Attempt Rate: Доля игроков, совершающих повторные попытки прохождения после откатов.
* Uninstalls (Удаления приложения): Динамика удалений игры в первые дни после запуска усложненного ивента.
